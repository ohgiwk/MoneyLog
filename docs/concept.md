# マネログ（MoneyLog） — コンセプト

## ひとことで言うと

> 「ルーチンをデザインして、お金の無駄をなくす家計簿」

ただ支出を記録するだけでなく、**繰り返しの支出を先に設計し、固定費を見直し続ける**ことで生活コストを最適化していく。

---

## ターゲット

- 自分用（個人利用）
- 固定費が膨らんでいる感覚はあるが、何にいくら払っているか把握できていない人
- 家計簿は続かないが、仕組み化・ルーチン化で生活を整えたい人

---

## コアコンセプト：3種類の出費

| 種別 | 説明 | 例 |
|------|------|-----|
| **固定費** | 毎月・毎年など一定サイクルで発生、金額がほぼ固定 | 家賃、サブスク、保険、スマホ代 |
| **変動ルーチン費** | 繰り返し発生するが金額が変動する | 食費、光熱費、交通費 |
| **臨時費** | 不定期・一時的な出費 | 旅行、医療費、家電の買い替え |

この3分類が マネログ（MoneyLog） の軸。すべての記録はどれかに属する。

---

## 主要機能

### 1. 固定費ダッシュボード
- 登録した固定費を一覧表示（月額換算で統一）
- 「必要 / 見直し中 / 解約済み」のステータス管理
- **節約額の累計・推移グラフ**：見直し前との差分をトラッキングしてモチベーション維持
- （将来）AIによる「代替サービス提案」「必要性スコア」アドバイス

### 2. 繰り返し出費のスケジューリング
- 繰り返しルールを登録（毎月・毎週・毎年など、日付・金額を指定）
- **自動反映モード**：期日になったら自動で記録に追加
- **手動確認モード**：期日になったら「確定しますか？」と通知 → ユーザーが確定
- カレンダー/タイムライン表示で「今月あと何が来るか」を把握

### 3. 臨時費の記録
- 通常の記録と分けて入力・表示
- タグ付け（旅行・医療・冠婚葬祭など）で後から集計可能
- 月次サマリーでルーチン費と臨時費を分けて表示

### 4. 月次サマリー
- 収支バランス（収入 - 固定費 - 変動費 - 臨時費）
- 先月・先々月との比較
- 節約進捗の可視化

## 画面構成（案）

```
📊 サマリー    — 月次レポート・収支推移
   ├ 📋 固定費 — 固定費一覧・節約トラッキング
   └ 🎯 目標   — 欲しいものリスト・貯金進捗
✏️ 記録        — 変動費・臨時費の入力
   └ 🛒 買い物メモ — 事前リスト作成・チェックして一括記録
📅 カレンダー  — 勤務日・日次支出・繰り返し出費の予定を一覧表示
```

### カレンダー画面の詳細

- 月カレンダーで勤務日（出勤🟢 / 休み）を管理
- 各日に日次支出合計を表示
- 繰り返しルールの予定日をカレンダー上に表示（未来の予定も見える）
- 日付タップで詳細（その日の収支一覧・勤務記録）を表示
- 月の収入予測・実績・支出合計をヘッダーにサマリー表示

## 現行 マネログ（MoneyLog） との違い

| 現行 | 新コンセプト |
|------|-------------|
| 過去の記録のみ | 過去 + 将来のスケジューリング |
| 固定費・変動費の区別なし | 3種分類を軸に設計 |
| 予算との比較 | 節約額の推移でモチベーション管理 |
| AIなし | AI節約アドバイス（将来） |

### 5. 買い物メモ

買い物前に「買うもの・予算」を登録しておき、買い物後にチェックするだけで記録が完了する機能。

- 買い物リストを事前作成（店名・予定日・アイテム＋予算額・カテゴリ）
- 買い物中にアイテムをチェック → 不要と判断したものは削除してその場で断捨離
- 買い物後に「記録する」で、チェック済みアイテムをtransactionsに一括登録
- 予算合計と実際の合計を比較して「予算内 / 超過」を表示
- よく買うものをテンプレート化して次回の入力を省略可能

### 6. 欲しいものリスト & 貯金目標

- 欲しいもの・買いたいものを金額とともに登録（例：カメラ ¥80,000）
- 複数登録して優先順位付け可能
- 毎月の収支・節約額から**貯金ペースを自動計算**し、「あと○ヶ月で買える」と表示
- 貯金目標を設定して進捗バーで可視化
- 節約で生まれた余剰金が目標達成を早める様子を視覚的に確認できる

### 6. 収入管理（オプション）

基本的にはシンプルに月収を登録するだけで十分。  
個人的ニーズとして**時給制ワーカー向けの収入予測機能**も持つ：

- 時給・想定稼働日数を登録 → その月の収入予測額を計算
- 実績（実際の稼働日数・支給額）を後から入力して予測と比較
- 収入が変動する月でも支出とのバランスをあらかじめ把握できる

> 将来的にはUI上でオン/オフ切り替え可能な「収入モード」として実装し、  
> 固定月収ユーザーは表示を簡略化できるようにする。
